*패킷 교환
- 이 용어는 컴퓨터 네트워크와 통신의 방식 중 하나로 현재 가장 많은 사람들이 사용하는 통신 방식이다.

- 작은 블록의 패킷으로 데이터를 전송하며 데이터를 전송하는 동안만 네트워크 지원을 사용하도록 하는 방법을 말한다.

- 장점으로는 네트워크 자원을 패킷 단위로 나누어 시간을 공유하므로 회선 효율성이 높다.

- 또한, 패킷 교환망은 Store-and-Forward 방식을 사용하기 때문에 데이터가 들어오는 속도와 나가는 속도를 맞출 필요 없이 

각 스테이션에 맞도록 속도를 조절할 수 있다. 이로써 전송 지연이 줄어들고 통신 안정성이 늘어난다.

- 예를 들면, 라우터와 스위치가 있다.


*human protocols
- 응답을 기대하면서 통신을 시작하는 시작점에 관련된 말이다.


*Network protocols
- 모든 인터넷에 일어나는 행위들이 프로토콜에 의해 관리되고 처리된다.


*ISP
- Internet Service Provider의 약자로, 개인이나 기업에게 인터넷 접속 서비스, 웹 사이트 구축 등을 제공하는 회사를 말한다.

- 예를 들면, Sk telecom, LG, KT 등등


*RFC
- Request For Comment의 약자로, 코멘트들의 요청문서이다.


*IETF
- Internet Engineering Task Force의 약자로, 코멘트들을 받아 수정하는 작업을 말한다.


*TCP/IP
- 회사마다 개별적인 네트워크만 사용하다가 최초로 네트워크 끼리 연결해주는 프로토콜이 개발되었다. 그것이 TCP/IP이다.

- TCP/IP는 인터넷 네트워크의 핵심 프로토콜이다. 인터넷에서 전송되는 정보나 파일들이 일정한 크기의 패킷들로 나뉘어

네트워크상 수많은 노드들의 조합으로 생성되는 경로들을 거쳐 분산적으로 전송되고, 수신지에 도착한 패킷들이 원래의 정보나

파일로 재조립되도록 하는 게 바로 TCP/IP의 기능이다.


*Ethernet
- 가까운 거리를 연결하는 근거리 통신망이다.

- 예를 드면, 학교나 회사 등등


*Network
- 독립적인 것들을 연결시키는 작업을 말한다.

- 예를 들면, 컴퓨터나 프린트를 말한다.


*Internet
- 네트워트 끼리 상호작용을 말한다.


*물리적 메체
- 두 지점 간의 통신을 위해 연결된 실체 매체이다.

- 예를 들면, 전화 회선, 동축 케이블, 마이크로파, 광섬유 등등이 있다.


*host
- 데이터의 패킷들을 보낸다

- 호스트의 주요 역할은 메세지를 패킷으로 쪼개서 access 네트워크로 내보내는 역할을 한다.
